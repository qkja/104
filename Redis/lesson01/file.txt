什么是 redis  是一个开源的,在内存中进行数据存储的

// 定义变量不就是在内存中存储数据吗?
// redis 是适合与分布式系统中才能发挥威力的
// 如果是单机的,那么直接用变量存储数据才是最优选择
// 进程具有独立性  --> 进程间通信
// 最长用的  是网络
// redis就是基于网络把自己内存的中数据给别的主机
// MySQL 是有一个问题的  就是访问速度有点慢
// redis 是因为是  快
// 和MySQL相比, 存储空间redis是有限的,也就是小
// 能不能又大又快 可以  把MySQL 和 Redis结合,就是cache技术
// 满足"二八原则"
// 但是凡事借由代价, 系统程度大大提升,如果数据发生变化,这里数据需要同步
// 没有银弹 
// streaming engin 最初的初心 最初是为作为一个消息中间件的(消息队列), 分布式下系统的生产者消费者的模型, 不过这个我们很少使用
